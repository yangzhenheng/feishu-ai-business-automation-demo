# 模型评测报告 Demo

该文档用于展示“模型选择不是凭感觉，而是按业务任务评估”的思路。

| 任务 | 模型族 | 输出质量 | 速度 | 成本 | 适合场景 |
|---|---|---:|---:|---|---|
| 业务日报 | Claude / GPT | 9 | 7 | 高 | 正式日报、管理层汇报、复杂总结 |
| 异常分类 | DeepSeek / Qwen | 8 | 8 | 低-中 | 批量结构化分类、风险打标、责任部门判断 |
| 中文客服 | Kimi / Qwen / GPT | 8 | 8 | 中 | 客户咨询、需求追问、线索摘要 |
| 代码生成 | Claude / GPT / Qwen Coder | 9 | 7 | 中-高 | 接口代码、自动化脚本、技术文档 |
| 本地SOP问答 | Ollama / 本地模型 | 6 | 6 | 低 | 内部资料、隐私数据、离线演示 |

## 评测维度

1. 输出是否稳定。
2. 是否能按 JSON / Markdown / 表格输出。
3. 中文表达是否自然。
4. 是否能处理业务上下文。
5. 成本是否适合高频调用。
6. 是否能作为备用模型。

## 结论

企业 AI 落地最关键的不是“哪个模型最强”，而是：

- 哪个任务需要强模型。
- 哪个任务可以用低成本模型。
- 哪些任务需要本地/私有化。
- 如何设置失败回退和人工兜底。
