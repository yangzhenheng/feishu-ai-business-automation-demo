# 女装电商业务闭环

场景：广州女装电商已经用飞书低代码跑通 ERP / WMS / CRM，现在希望逐步接入 AI、云函数、审批流、物流/店铺/财务 API、RAG 和预测模型。

## 业务链路

客户下单 -> 店铺订单同步 -> ERP订单池 -> WMS库存校验 -> 发货/物流跟踪 -> 客服售后 -> 财务回款 -> AI日报/异常预警

## 关键节点

| 节点 | 数据对象 | 负责人 | 自动化价值 |
|---|---|---|---|
| 客户下单 | 订单、SKU、买家备注、支付金额 | 店铺运营 | 多平台订单标准化。 |
| 店铺订单同步 | 订单同步记录 | 低代码开发工程师 | 云函数定时拉取抖音、天猫、拼多多订单。 |
| ERP订单池 | 订单状态、负责人、异常标签 | 运营 / 客服 | 形成统一订单视图。 |
| WMS库存校验 | 可用库存、安全库存、缺口数量 | 仓储 / 采购 | 低库存自动触发补货审批。 |
| 物流跟踪 | 运单号、物流状态、异常原因 | 仓储 / 客服 | 延迟、退回、少发漏发自动预警。 |
| 客服售后 | 售后单、话术、客户诉求 | 客服 | RAG 提供尺码、退换货和商品知识回复。 |
| 财务回款 | 订单金额、实收金额、差异原因 | 财务 | 自动定位平台扣点、退款和优惠券差异。 |
| AI日报 | 经营摘要、风险、建议动作 | 管理层 | 每天固定输出业务复盘和下一步动作。 |

## 面试讲解重点

这个项目展示的不是单个接口，而是把飞书多维表格、审批流、云函数、Webhook、SQL 和 AI 串成业务闭环。岗位落地时，先用低代码快速打通可视化和协同，再逐步把高频判断交给 AI 和预测模型。
