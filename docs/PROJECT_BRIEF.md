# 项目说明书：AI + 飞书低代码业务自动化系统

## 1. 项目定位

本项目用于展示 AI 工具如何接入企业业务流程。项目不是重型 ERP，也不是算法研究，而是一个 **能快速演示、能快速扩展、能用于面试讲解的低代码业务自动化 MVP**。

目标岗位方向：飞书低代码实施、AI业务自动化、供应链数字化、企业AI工具落地、AI客服/机器人自动化。

## 2. 业务痛点

很多中小企业和业务团队仍然依赖以下方式管理流程：

- Excel 记录订单、库存、异常。
- 微信群/飞书群人工催办。
- 客户咨询散落在聊天记录里，无法统一沉淀。
- 异常问题没有闭环，责任人和截止时间不清晰。
- 日报靠人工整理，重复、慢、容易漏。

## 3. 解决方案

用低代码和 AI 做一个轻量业务系统：

1. 用数据表承载订单、库存、异常、客户线索、待办事项。
2. 用自动化规则识别库存不足、交期风险、未完成待办。
3. 用 Webhook 把关键事件推送到飞书/企业微信群。
4. 用 AI 生成日报、总结异常、提出处理优先级。
5. 用机器人命令让业务人员快速查询 `/日报`、`/异常`、`/库存`、`/客户线索`。
6. 用 AI 客服承接网站咨询，自动沉淀客户线索。

## 4. 系统模块

### 4.1 订单协同模块

字段设计：订单编号、客户、SKU、数量、交期、负责人、状态、创建时间。

目标：让业务人员能追踪订单状态，识别异常订单。

### 4.2 库存预警模块

字段设计：SKU、商品名称、当前库存、安全库存、供应商、仓库。

目标：当当前库存低于安全库存时，自动进入预警列表。

### 4.3 异常闭环模块

字段设计：异常编号、订单编号、异常类型、原因、负责人、优先级、状态、创建时间。

目标：把“口头反馈”变成可追踪的异常单。

### 4.4 AI日报模块

输入：订单总数、异常列表、低库存SKU、待办、客户线索。

输出：今日业务日报、重点风险、优先处理事项、AI处理建议。

### 4.5 机器人指令模块

支持命令：

- `/日报`：生成今日业务日报
- `/异常`：查看未关闭异常
- `/库存`：查看库存预警
- `/客户线索`：查看新客户咨询
- `/待办`：查看未完成事项

### 4.6 AI客服模块

支持场景：

- 服务介绍
- 报价咨询
- 网站AI客服咨询
- 飞书/企业微信自动化咨询
- 客户需求收集
- 线索沉淀

## 5. 技术实现

- 后端：FastAPI
- 数据库：SQLite
- 前端：HTML + CSS + JavaScript
- 通知：飞书自定义机器人 Webhook / 企业微信机器人 Webhook
- AI：OpenAI-Compatible API，可无Key回退到规则日报
- 部署：本地运行 / Docker / 服务器部署

## 6. 可扩展到真实业务的方式

当前项目使用 SQLite 模拟数据表。真实落地时可以替换为：

- 飞书多维表格作为业务数据库
- 飞书审批流作为异常审批节点
- 飞书自动化规则作为状态触发器
- 企业微信/飞书机器人作为通知入口
- 知识库 + RAG 作为 AI 客服能力
- 服务器部署作为 24 小时运行环境

## 7. 面试价值表达

这个项目能证明：

1. 能理解业务流程，不是只会聊 AI 概念。
2. 能把流程拆成表结构、规则、通知、日报。
3. 能用基础代码和 Webhook 把系统串起来。
4. 能做出可演示的 MVP。
5. 能和飞书低代码、企业微信、AI客服、本地部署等方向衔接。
