# 飞书审批流设计

## 库存不足 -> 采购补货审批

触发条件：`available_stock < safety_stock`。

审批字段：SKU、商品名、仓库、当前可用库存、安全库存、缺口数量、建议补货数量、供应商、预计到货时间。

审批人：仓储负责人 -> 采购负责人 -> 运营负责人。

## 交期风险 -> 负责人升级审批

触发条件：订单临近承诺发货时间，且库存不足或物流揽收延迟。

审批字段：订单号、店铺、买家备注、异常原因、当前负责人、建议升级负责人。

动作：创建 P1 异常，推送飞书群，超过 24 小时未处理则升级。

## 高意向客户 -> 销售跟进任务

触发条件：CRM 线索 `intent_level = 高`。

任务字段：客户来源、需求摘要、联系方式、推荐负责人、截止跟进时间。

动作：创建销售任务，并在 AI 日报中展示未跟进高意向客户。

## 财务差异 -> 财务复核审批

触发条件：`diff_amount <> 0`。

审批字段：订单号、店铺、订单金额、已回款金额、差异金额、差异原因、关联售后单。

动作：财务复核后标记为正常扣点、待运营确认、待平台结算或异常追款。
